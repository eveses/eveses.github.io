**포트 스캐닝과  Nmap**

## 포트 스캐닝이란?

포트 스캐닝이란 특정 호스트의 TCP/UDP 포트들의 상태를 파악하기 위한 정보 수집 행위이다. <br> 
모의 해킹의 필수적인 첫 단계로 이러한 행위를 통하여 공격자는 어떻게 공격을 진행할지 파악할 수 있다.<br><br> 
포트 스캐닝 방법은 크게 TCP 포트 스캐닝과 UDP 포트 스캐닝으로 나뉜다. <br><br>
TCP 스캔은 TCP-connect 스캔과 TCP-syn 스캔으로 나뉘는데 <br>connect 스캔은 TCP 3 way hadshake 과정을 모두 진행한 후 포트의 상태를 확인하는 것이고 <br>syn 스캔은 TCP 3 way hadshake 과정중 절반만 사용하여 포트의 상태를 확인하는것이다. <br> 
따라서 각각 장단점이 있으므로 상황에 맞춰 사용해야 한다. <br><br>
UDP 스캔은 상태가 존재하지 않는 연결이기 때문에 TCP와는 다르게 패킷을 전송하고 응답이 돌아오냐 아니냐에 따라 상태 여부를 결정한다.

## Nmap 이란?

Nmap (Network Mapper) 이란 오픈소스 네트워크 탐지 도구로 포트 스캐닝에 가장 많이 사용되는 툴 이다.<br><br>
Nmap 공식 사이트 [https://nmap.org/](https://nmap.org/) 에서 많은 정보를 찾아 볼 수 있다. 

## Nmap 사용법

대표적인 TCP-connect 스캔, TCP-syn 스캔, UDP 스캔을 각각 어떻게 하는지 알아보겠다.<br><br>
먼저 TCP-connect 스캔은 Nmap의 가장 기본적인 스캔으로 ip주소만 주면 된다.<br> (root유저일경우 기본적으로 syn스캔, 일반유저일경우 기본적으로 connect스캔)
```
nmap <ip>
```
다음으로 TCP-syn 스캔의 경우 -sS 플래그를 주고 root권한이 필요하므로 sudo를 붙여야 한다.
```
sudo nmap -sS <ip>
```
마지막으로 UDP 스캔의 경우에도 root권한이 필요하고 -sU 플래그를 붙이면 된다.
```
sudo nmap -sU <ip>
```
이들이 Nmap의 기본적인 사용법이고 이에 여러가지 플래그들을 추가하여 더 많은 방법으로 사용할 수 있다.
## 자주 사용되는 Nmap 플래그

| 옵션          | 설명                                                                       |
|---------------|----------------------------------------------------------------------------| 
| `-Pn`         | Ping 스캔을 통한 호스트 확인(host discovery) 비활성화                       |
| `-n`          | DNS 해석 비활성화 (ip 주소로 스캔할때 유용)                                 |
| `-sS`         | SYN 스캔                                                                    |
| `-sT`         | CONNECT 스캔                                                              |
| `-sU`         | UDP 스캔                                                                   |
| `-p <포트>`   | 특정 포트 스캔 (여러 포트 사용 가능)                                          |
| `-O`          | 운영 체제 감지                                                              |
| `-sV`         | 서비스 및 버전 감지                                                         |
| `-sC`         | 기본 스크립트 사용하여 더 자세히 스캔                                         |
| `-T`          | 스캔 속도 조정 (T1 부터 T5 까지)                                              |
| `-oA <파일명>`| 결과를 여러 형식으로 저장                                                      |
| `--open`      | 열린 포트만 표시                                                              |
| `--top-ports` | 가장 많이 사용되는 포트들을 스캔함                                             |
