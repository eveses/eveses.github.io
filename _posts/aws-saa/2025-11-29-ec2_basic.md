## EC2 - Elastic Compute Cloud

EC2는 AWS에서 제공하는 IaaS이고, 가장 많이 사용되는 서비스중 하나이다.

OS, CPU, RAM, 네트워크, 보안그룹, 저장소, 스크립트 등 다양한 커스터마이징이 가능하다.

**[EC2 Instance Type]**

- Compute Optimized : CPU 성능 위주, 고성능 연산
- Memory Optimized : 대용량 메모리 위주, 캐시 서버, 인메모리 DB
- Storage Optimized : 고성능 저장소 I/O 위주, DB, 파일시스템
- General Purpose : 균형 잡힌 성능, 일반적

**[BootStrap]**

EC2를 처음 생성할때 자동으로 실행되는 커맨드를 말한다. 초기 생성 시 User Data를 이용하여 스크립트를 추가할 수 있고 소프트웨어 업데이트, 다운로드, 초기 설정등에 유용하다.

**[Security Group]**

보안그룹으로 AWS에서 사용하는 일종의 방화벽이다. 

- 허용할 규칙만 추가 가능
- IP나 또다른 보안그룹을 참조
- 여러개의 인스턴스에 부착 가능
- 특정 VPC에 종속

**[연결방법]**

- 기본적으로 SSH를 이용하여 연결, 구형 윈도우의 경우 putty 사용. 

- EC2 Instance Connect를 이용하여 따로 CLI나 키 없이 연결 가능하지만 특정 AMI에서만 가능.

- 세션 매니저 역할을 부여한 경우 세션 매니저를 통해 접근 가능.

**[EC2 Purchasing Options]**

- On-Demand : 정확히 사용한만큼만 돈을 내는 구조, 가장 비쌈
- Reserved : 특정 기간동안 (1 or 3년) 약정하는 구조, 일부 옵션에 따라 인스턴스 페밀리 변경 가능
- Saving Plan : 시간당 지불 금액을 약정, 가장 유연한 비용 절감, 추가되는 비용은 On-Demand로 처리
- Spot Instances : 가장 저렴하지만 언제든지 인스턴스가 사라질 수 있음, 일회성 작업 용이
- Dedicated Hosts : AWS 물리 서버 자체를 빌리는것, 소프트웨어나 라이선스를 따로 사용해야 할 경우, 가장 비쌈
- Capacity Reservation : 용량 예약, 원할때 바로 빌릴 수 있도록

*Spot Fleets - 스팟 인스턴스 관리 매니저 (On-Demand도 사용 가능)  
전략 : Lowest Price(가장 저렴하도록), Diversified(분산), Capacity Optimized(재고가 널널하도록), Price and Capacity Optimized(비용+재고)

**[Placement Group]**

EC2의 물리적 배치를 설정하는 기능, 3가지 전략

- Cluster : 동일 AZ, 최대한 가깝게, 고성능, 저지연
- Spread : 최대한 분산, 장애 위협 최소화, AZ별 최대 7개까지만 가능
- Partition : 각각 다른 파티션(랙)에 분산, 수백개까지 가능, 대량 분산 시스템

**[EC2 Hibernate]**

기본적으로 EC2는 Stop, Terminate가 있는데 Stop시 Hibernate 되도록 설정 가능

- Stop : 인스턴스를 잠시 중지, 데이터 유지, 메모리 삭제
- Stop(Hibernate): 인스턴스를 중지, 데이터 유지, 메모리 유지
- Terminate : 인스턴스 종료, 데이터 삭제(EBS 옵션에 따라 유지 가능), 메모리 삭제
