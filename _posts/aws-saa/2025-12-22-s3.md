## 1. 기본 구조

<h3>Buckets & Objects</h3>

* **Buckets:** 객체를 담는 컨테이너로 이름은 **고유**해야 하며, 생성 시 특정 리전 선택.
* **Objects:** 데이터의 기본 단위.
    * **Key:** 객체의 전체 경로(예: `images/logo.png`). S3는 실제 폴더 구조가 아닌 키-값 기반의 구조이지만, `/`를 사용하여 계층 구조처럼 보이게.
    * **Value:** 데이터 본체 (최대 5TB).
    * **Metadata:** 시스템 메타데이터(크기, 수정일)와 사용자 정의 메타데이터.
* **Strong Consistency:** 쓰기/수정/삭제 직후 조회해도 즉시 최신 데이터 반영.

## 2. 데이터 보호 및 복제

<h3>Versioning (버전 관리)</h3>

* 실수로 데이터를 덮어쓰거나 삭제하는 것을 방지하기 위한 기능.
* 삭제 시 실제 데이터가 지워지는 대신 **Delete Marker**가 붙으며, 이를 제거하면 복구가 가능.
* **MFA Delete:** 버전 삭제 시 추가 인증(OTP 등)을 강제하여 보안을 극대화.

<h3>Replication (CRR & SRR)</h3>

* **CRR (Cross-Region Replication):** 법적 규정 준수나 재해 복구(DR)를 위해 다른 리전으로 복제.
* **SRR (Same-Region Replication):** 테스트/운영 환경 분리나 로그 통합을 위해 같은 리전 내 다른 버킷으로 복제.  
***중요:** 복제 기능을 쓰려면 원본과 대상 버킷 모두 버전 관리가 켜져 있어야 함.

## 3. 스토리지 클래스와 수명 주기

<h3>Storage Classes (스토리지 등급)</h3>

| 등급 | 특징 | 주요 용도 |
| :--- | :--- | :--- |
| **Standard** | 높은 가용성, 빠른 액세스 | 자주 사용하는 활성 데이터 |
| **Intelligent-Tiering** | 접근 패턴을 분석해 자동 이동 | 패턴을 모를 때 최적 (자동 비용 절감) |
| **Standard-IA** | 낮은 저장 비용, 높은 인출 비용 | 가끔 접근하지만 즉시 필요할 때 (백업 등) |
| **One Zone-IA** | 1개 AZ에만 저장 (비용 20% 저렴) | 재현 가능한 보조 데이터 |
| **Glacier Instant** | 저렴한 비용, 밀리초 단위 인출 | 아카이브 중 가끔 조회가 필요할 때 |
| **Glacier Flexible** | 매우 저렴, 1분~12시간 인출 | 일반적인 장기 백업 |
| **Glacier Deep Archive** | 최저 비용, 12~48시간 인출 | 규정 준수를 위한 장기 보관 (거의 안 꺼냄) |

<h3>Lifecycle Rules & Analytics</h3>

* **Lifecycle Rules:** 
    * **Transition:** 일정 기간(예: 30일) 후 더 저렴한 등급으로 이동.
    * **Expiration:** 일정 기간 후 객체 영구 삭제.
* **S3 Analytics:** 사용자의 데이터 접근 패턴을 분석하여 어떤 데이터를 IA 등급으로 옮기면 비용이 절감될지 보고서 제공.

## 4. 보안 및 관리 기능 

<h3>Security & Access Control</h3>

* **Bucket Policy:** 버킷 전체에 적용되는 정책. 특정 IP, 특정 VPC에서만 접근하게 할 때 유용. (IAM Policy: 사용자/역할 기준, Bucket Policy: 리소스 기준)
* **Object Lock:** WORM(Write Once, Read Many) 방식. 특정 기간 동안 데이터를 절대 지우거나 고칠 수 없게 고정. (버킷 생성시에만 설정 가능)
* **Block Public Access:** 클릭 한 번으로 버킷이 외부에 공개되는 것을 차단.
* **Request Pays (요청자 지불):** 데이터 다운로드 비용을 버킷 소유자가 아닌 다운로드하는 사람이 내도록 설정.

<h3>Encryption(암호화)</h3>

* **SSE-S3:** S3 기본 암호화 (AWS 관리 키).
* **SSE-KMS:** KMS 서비스와 통합. 누가 언제 암호화했는지 감사(Audit) 가능.
* **SSE-C:** 사용자가 직접 키를 제공.

## 5. 성능 및 대규모 작업 

<h3>S3 Performance Optimization</h3>

* **Prefix 확장성:** S3는 Prefix당 초당 3,500회(쓰기), 5,500회(읽기) 요청 처리. 성능이 부족하면 폴더(Prefix)를 더 세분화.
* **Multipart Upload:** 파일을 조각내어 병렬로 처리. 100MB 이상 권장, 5GB 이상 필수.
* **Transfer Acceleration:** 전 세계 엣지 로케이션을 통해 장거리 업로드 속도 개선.

<h3>Batch Operations & Storage Lens</h3>

* **Batch Operations:** 수백만 개의 객체에 대해 복사, 태그 지정, Lambda 실행 등을 한 번에 수행. (작업 일람표인 Manifest 파일 필요)
* **Storage Lens:** 조직 내 전체 버킷 사용 현황, 비용 효율성, 보안 상태를 시각화 대시보드로 제공.

## 6. 추가 핵심 기능 

<h3>Event Notification & Static Web</h3>

* **Event Notification:** 파일 업로드/삭제 발생 시 Lambda, SQS, SNS를 호출하여 워크플로우를 자동화. (예: 영상 업로드 시 인코딩 시작)
* **Static Web Site:** S3 버킷을 웹 서버처럼 사용하여 HTML/JS 웹사이트를 호스팅. 저렴하고 관리가 필요 없음.

<h3>S3 Select</h3>

* 객체(CSV, JSON 등) 전체를 받지 않고, 필요한 부분만 SQL로 필터링해서 가져와 속도를 높이고 비용 절감.
